1.
 
Introduction
 
Formula
 
1
 
race
 
outcomes
 
depend
 
on
 
many
 
interacting
 
factors,
 
such
 
as
 
driver
 
performance,
 
team
 
capability,
 
weather
 
conditions,
 
tire
 
strategies,
 
and
 
qualifying
 
positions.
 
Because
 
these
 
factors
 
interact
 
in
 
complex
 
ways,
 
predicting
 
race
 
outcomes
 
is
 
a
 
challenging
 
problem
 
that
 
can
 
benefit
 
from
 
machine
 
learning
 
techniques.
 
The
 
goal
 
of
 
this
 
project
 
is
 
to
 
develop
 
an
 
AI-based
 
system
 
that
 
predicts
 
the
 
probability
 
of
 
a
 
driver
 
winning
 
a
 
Formula
 
1
 
race
 
using
 
historical
 
race
 
data.
 
The
 
project
 
will
 
utilize
 
the
 
FastF1
 
Python
 
library
,
 
which
 
provides
 
access
 
to
 
detailed
 
race
 
timing
 
data,
 
session
 
information,
 
lap
 
times,
 
and
 
telemetry
 
from
 
Formula
 
1
 
events.
 
The
 
key
 
idea
 
behind
 
this
 
project
 
is
 
that
 
different
 
race
 
parameters
 
influence
 
the
 
outcome
 
with
 
different
 
levels
 
of
 
importance.
 
For
 
example,
 
qualifying
 
positions
 
may
 
strongly
 
affect
 
race
 
results
 
because
 
drivers
 
starting
 
near
 
the
 
front
 
have
 
a
 
strategic
 
advantage.
 
Similarly,
 
weather
 
conditions
 
such
 
as
 
rain
 
may
 
significantly
 
influence
 
race
 
outcomes
 
due
 
to
 
changes
 
in
 
tire
 
strategy
 
and
 
driving
 
difficulty.
 
Instead
 
of
 
manually
 
assigning
 
importance
 
to
 
these
 
variables,
 
machine
 
learning
 
models
 
will
 
be
 
used
 
to
 
automatically
 
learn
 
the
 
relative
 
contribution
 
of
 
each
 
feature
 
based
 
on
 
historical
 
race
 
data.

●

●
 
To

5.
 
Machine
 
Learning
 
Models
 
The
 
project
 
will
 
explore
 
multiple
 
machine
 
learning
 
models
 
for
 
predicting
 
race
 
winners.
 
Logistic
 
Regression
 
(Baseline
 
Model)
 
Logistic
 
regression
 
will
 
be
 
used
 
as
 
a
 
baseline
 
model.
 
It
 
is
 
simple,
 
interpretable,
 
and
 
allows
 
us
 
to
 
directly
 
analyze
 
the
 
learned
 
weights
 
associated
 
with
 
each
 
feature.
 
Gradient
 
Boosted
 
Trees
 
(Primary
 
Model)
 
Gradient
 
boosting
 
models
 
such
 
as
 
XGBoost
 
or
 
LightGBM
 
will
 
be
 
used
 
as
 
the
 
primary
 
prediction
 
models.
 
These
 
models
 
are
 
well-suited
 
for
 
structured
 
tabular
 
data
 
and
 
can
 
capture
 
complex
 
nonlinear
 
relationships
 
between
 
features.
 
Gradient
 
boosting
 
models
 
also
 
provide
 
feature
 
importance
 
measures,
 
allowing
 
us
 
to
 
identify
 
which
 
variables
 
have
 
the
 
strongest
 
impact
 
on
 
race
 
outcomes.
 
Random
 
Forest
 
(Optional
 
Comparison
 
Model)
 
Random
 
Forest
 
may
 
be
 
used
 
as
 
an
 
additional
 
model
 
for
 
comparison
 
to
 
evaluate
 
different
 
ensemble
 
learning
 
techniques.
 
 
6.
 
Model

The

will

sets
 
based
 
on
 
race

to
 
avoid

Example
 
split:
 
●
 
Training
 
data:
 
seasons
 
2018–2023
 
 
●
 
Validation
 
data:
 
season
 
2024
 
 
●
 
Test
 
data:
 
season
 
2025
 
 
This
 
approach
 
ensures
 
that
 
the
 
model
 
is
 
evaluated
 
on
 
future
 
races
 
that
 
were
 
not
 
used
 
during
 
training.
 
Data
 
preprocessing
 
steps
 
will
 
include:
 
●
 
Handling
 
missing
 
values
 
 
●
 
Encoding
 
categorical
 
variables
 
(drivers,
 
teams,
 
circuits)
 
 
●
 
Feature
 
normalization
 
where
 
required
 
 
 
7.
 
Evaluation
 
Metrics
 
Since
 
only
 
one
 
driver
 
wins
 
each
 
race,
 
the
 
dataset
 
will
 
be
 
highly
 
imbalanced.
 
Therefore,
 
multiple
 
evaluation
 
metrics
 
will
 
be
 
used.
 
Evaluation
 
metrics
 
will
 
include:
 
●
 
Precision
 
 
●
 
Recall
 
 
●
 
F1-score
 
 
●
 
ROC-AUC
 
 
●
 
Log-loss
 
 
Additionally,
 
we
 
will
 
evaluate
 
whether
 
the
 
model
 
places
 
the
 
actual
 
race
 
winner
 
among
 
the
 
top
 
predicted
 
probabilities,
 
which
 
is
 
important
 
for
 
practical
 
prediction
 
performance.

By

how

patterns

The

could

insights

which
 
factors
 
most
 
strongly
